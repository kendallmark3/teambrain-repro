15–25 min

Intent-Driven Development

❌ Old Way

"Write me an AI endpoint for user chat."
  • Vague — no context or constraints
  • No validation rules specified
  • No security requirements stated
  • Produces hallucinated implementation

✅ Intent-Driven

Create POST /api/v1/chat Business goal: stream a Claude response to an authenticated user's prompt. API contract: accept {message, history}, return streamed text chunks. Validation: message non-empty, max 4096 tokens. 400 if exceeded. Security: API key from env only. Never log raw prompt contents. Observability: log token usage + latency. Arch: routes → service → LLM client.
Intent = Full Context

What Goes Into an Intent

🎯

Business Goal

Why does this feature exist? What user problem does it solve? State it in one sentence.

📡

API Contract

Exact method, path, request shape, response DTO, and pagination model.

✅

Validation Rules

Input constraints, field formats, boundary values, and what returns 400.

🔒

Security Constraints

Auth requirements, role checks, ownership validation, and what returns 403.

📊

Observability

What gets logged, what gets metrics, what gets traced, and where.

🏗️

Arch Boundaries

Which layers are involved, which services to call, and what must not be crossed.

25–35 min

The CLEAR Prompt Method

C — Context: Python 3.11, FastAPI, Anthropic SDK, Pydantic, Postgres. L — Layers: Route → Service → LLM Client. No prompt logic in handlers. E — Endpoint: POST /api/v1/completions — stream a Claude chat response. A — Assertions: Message non-empty, model in allowed list. 400 if invalid. R — Risk flags: Rate limit per user. Never expose raw API keys in responses.

"Good prompting reduces rework by 50%."

35–45 min

Claude.md — The Governance Layer

What It Is

  • A markdown file committed to your repo root
  • Loaded automatically by every Claude Code session
  • Acts as standing instructions for the AI
  • Shared across your entire team automatically

What It Enforces

  • No any — strict types enforced on every prompt
  • Sensitive path rules (auth, payments, migrations)
  • Folder structure and naming conventions
  • Required error handling and logging patterns

❌ Without Claude.md

  • Claude invents its own patterns
  • Every session starts from scratch
  • No enforcement of team standards
  • Inconsistent, untraceable outputs

✅ With Claude.md

  • Standards enforced on every prompt
  • Team conventions are non-negotiable
  • New devs inherit standards instantly
  • Auditable, repeatable, governed output
45–52 min

Guardrails You Must Keep

🚫

No Secrets

Never commit API keys, passwords, or tokens. Validate via env vars at startup only.

👀

Review Everything

Read every line Claude generates before merging. You own the output — not the model.

🧪

Validate Edge Cases

Claude optimizes for the happy path. Ask explicitly for boundary and error cases.

⚠️

Watch Hallucinations

Verify library APIs, method signatures, and version compatibility against real docs.

✅

Run Tests

Never ship AI-generated code without passing your full test suite. Tests are not optional.

"AI writes fast. Engineers validate."
